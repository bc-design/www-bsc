2016-04-23 21:29
Last night I stayed up until 06:00 dissecting an "8-Bit LED Digital Tube Module" I bought from China on Ebay for a buck

brandon
2016-04-23 21:30
I've done this with a couple devices in an effort to better understand how hardware is put together and how firmware interacts with it

brandon
2016-04-23 21:32
it consists of two serial-to-parallel latched shift registers connected end-to-end

brandon
2016-04-23 21:32
wired to 2x 4-digit, 7-segment (plus dot) LED displays

brandon
2016-04-23 21:33
There are two clock inputs, SLCK (shift register clock) and RCLK (storage register clock)

brandon
2016-04-23 21:34
a serial input, a serial output, 8x parallel outputs, power, ground, and a storage-register-clear pin

brandon
2016-04-23 21:37
the shift register consists of 8x flip-flops connected end-to-end to eachother and in parallel to SCLK

brandon
2016-04-23 21:40
when SCLCK cycles high, the digital value in flip-flop 'n' is shifted into flip-flop 'n+1', and the digital value at the serial input is shifted into the first flip-flop

brandon
2016-04-23 21:41
each flip-flop's current output value is accessible on one of the 8x parallel output pins

brandon
2016-04-23 21:42
so the data comes in serially and is multiplexed in parallel to the outputs by way of these sequential, clock-controlled bit shifts

brandon
2016-04-23 21:43
but it's a LATCHED shift register, so there's one more layer of complexity

brandon
2016-04-23 21:44
in this configuration, each flip-flop's current output value is actually connected to a corresponding latched flip-flop in a storage register

brandon
2016-04-23 21:46
and NOT directly to the 8x parallel output pins

brandon
2016-04-23 21:46
the storage register latched flip-flops will only take on the values in the shift register flip-flops when RCLK is cycled high

brandon
2016-04-23 21:47
why this is desirable for this (and many other) multiplexing application:

brandon
2016-04-23 21:49
the shift register holds 8 bits == 1 byte, 0b00000000

brandon
2016-04-23 21:50
in this device, the parallel output pins are connected to corresponding segments on the LED display

brandon
2016-04-23 21:50
the pattern of bits in the shift register set which segments are lit

brandon
2016-04-23 21:51
(somewhat confusingly, '0' is lit and '1' is unlit in this configuration, but I'll get to that)

brandon
2016-04-23 21:51
for instance, 0b10000000 is '8' on the display

brandon
2016-04-23 21:52
0b00000000 is '8.' on the display

brandon
2016-04-23 21:53
0b00000010 is '6.' on the display

brandon
2016-04-23 21:53
0b10000010 is '6' on the display

brandon
2016-04-23 21:54
you can cycle through various input values and figure out which LED elements correspond to which bits

brandon
2016-04-23 21:55
so, if the parallel outputs were directly connected to the shift register flip-flops, in the process of shifting in a new set of bits to display a particular character, the LED elements would flicker as the bits are sequentially shifted into position

brandon
2016-04-23 21:57
instead, the desired bit sequence is presented at the serial input while SCLK is cycled until the desired bit pattern is in position in the shift register

brandon
2016-04-23 21:58
then, RCLK is cycled to copy the desired bit pattern from the shift register into the storage register

brandon
2016-04-23 21:59
so you can 'snap' from displaying one character to displaying another

brandon
2016-04-23 22:01
so given all of this, you now have a way to deliver a bit sequence to a shift register, copy if to the storage register, and display it on a single 7-segment LED module

brandon
2016-04-23 22:02
but how to display a sequence of characters on 8x 7-segment LED modules?

brandon
2016-04-23 22:02
the answer is, with another shift register!

brandon
2016-04-23 22:02
you connect another shift register's input to the first shift register's output

brandon
2016-04-23 22:03
bits that 'fall off' the first shift register's last flip-flop are shifted into the second register's first flip-flop

brandon
2016-04-23 22:04
connect thee two shift registers' RCLK and SCLK in parallel, and you now have one big 16-bit flip flop

brandon
2016-04-23 22:05
the first shift register was hooked to the segments of a single 7-segment LED display module

brandon
2016-04-23 22:05
specifically, hooked to the GROUND pins of those segments (so '0' in the shift register = 'on')

brandon
2016-04-23 22:09
(this is a 'common anode' configuration of the LED segments in a module, where all of the segments in a module share a power (anode) pin and each segment is individually grounded... each ground hooked to a parallel output pin on the shift register!)

brandon
2016-04-23 22:13
now, hook 7 more of those modules to the first shift register, in parallel

brandon
2016-04-23 22:15
and hook the common power (anode) pins of all segments to the Vin power rail

brandon
2016-04-23 22:15
now if you shift an '8' into the shift register and copy it to the parallel outputs, all 8x of the modules will display an '8'

brandon
2016-04-23 22:16
disconnect the common anode pins from the power rail and instead, connect them to the parallel outputs of the SECOND SHIFT REGISTER

brandon
2016-04-23 22:17
if a parallel output on the second shift register is '1', the corresponding LED module is powered and its elements WILL light up according to the bit pattern presented in the first shift register

brandon
2016-04-23 22:18
if a parallel output on the second shift register is '0', that LED module will stay dark

brandon
2016-04-23 22:19
so now you're shifting in 16 bits (2 bytes) at a time: the first bit describes which modules will be displaying the character, and the second bit describes which elements will be lit on the selected modules

brandon
2016-04-23 22:20
for instance, shift in 0b10101010 followed by 0b10000010 to display '6' on every other module
